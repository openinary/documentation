## Quick Start

Run Openinary locally with Docker in just a few commands. Openinary supports two deployment modes depending on your needs.

<Steps>

  <Step title="Clone the repository">

```bash
git clone https://github.com/openinary/openinary.git
```

  </Step>

  <Step title="Change into the project directory">

```bash
cd openinary
```

  </Step>

  <Step title="Configure environment variables (optional)">

  <Tip>
  For local testing with default settings (using local file storage), you can skip this step.
  </Tip>

  To configure S3-compatible cloud storage (Cloudflare R2 or AWS S3), copy the example environment file:

  ```bash
  cp apps/api/.env.example apps/api/.env
  ```

  Then edit `apps/api/.env` with your cloud storage credentials. For full details and options, see the [Storage configuration](/configuration) guide.


  </Step>

  <Step title="Choose your deployment mode">

  Select the deployment mode that fits your use case:

  <Tabs>

  <Tab title="Mode 1: API Only">

  Deploy only the Hono API server for backend services and integrations.

  ```bash
  # Start API only
  docker compose --profile api up --build

  # Run in background
  docker compose --profile api up -d --build
  ```

  **Access**: API available at http://localhost:3000

  </Tab>

  <Tab title="Mode 2: API + Dashboard">

  Deploy the complete stack (API + Next.js frontend) with nginx as a reverse proxy. All traffic goes through port 3000:

  - `/` → Next.js frontend
  - `/api` → Hono API

  ```bash
  # Start api + dashboard
  docker compose --profile full up --build

  # Run in background
  docker compose --profile full up -d --build
  ```

  **Access**: Application available at http://localhost:3000

  </Tab>

  </Tabs>

  </Step>

</Steps>

## Stopping the Application

To stop services, use the same profile you started with:

<Tabs>

<Tab title="API Only">

```bash
docker compose --profile api down
```

</Tab>

<Tab title="API + Dashboard">

```bash
docker compose --profile full down
```

</Tab>

</Tabs>

<Tip>
Add the `-v` flag to also remove volumes and clear cache: `docker compose --profile api down -v`
</Tip>

## Development Mode (Alternative)

If you prefer to run in development mode without Docker:

```bash
# Install dependencies
pnpm install

# Start development servers
pnpm dev
```

Development URLs will be the same (API: http://localhost:3000, Frontend: http://localhost:3001)

---

<Warning>
**Production Deployment**: When deploying to production, always use cloud storage (S3 or R2) instead of local file storage. Configure your cloud storage credentials in `apps/api/.env`. The local `public/` folder is only suitable for testing and development.
</Warning>
